# RTLX 15.2.0 — Browser Survival Hardening

RTLX 15.2 is a reliability-only update over 15.1. It does not change product scope or Profile Schema v3.

## Added

- durable checksummed local write-ahead storage journal with restart/update recovery;
- trusted-context storage access and deterministic quota governor;
- single-flight background initialization and race-safe content-script reconciliation;
- document/runtime epoch validation and stale-message rejection;
- discarded-tab intent recovery and bounded multi-tab broadcast;
- canonical JSON message contracts, payload limits, response validation, and deadlines;
- persistent alarm leases, missed-run detection, structured retries, and bounded circuit state;
- cross-browser evidence, worker-termination, and bounded soak harnesses;
- separate Chromium, Edge, and Firefox evidence CI jobs.

## Toolchain security

Vitest and `@vitest/coverage-v8` were upgraded to 4.1.8 with the official single-worker migration (`maxWorkers: 1`, `isolate: false`). The final production and complete npm audits both report zero vulnerabilities.

## Preserved

Profile Schema v3, Persian/English scope, semantic `dir`, Vazirmatn/Inter policy, Amazon Ember local-only policy, protection zones, privacy defaults, optional host access, declarative profiles, and journal-safe ownership-checked rollback.

## Executed development evidence

- 63 test files and 163 tests pass.
- Coverage passes configured thresholds: 85.43% statements, 75.05% branches, 90.20% functions, and 87.03% lines.
- Chromium, Edge, and Firefox builds, direct Chromium runtime smoke, manifest validation, Firefox lint, security scan, and both dependency audits pass.
- Two release runs produce byte-identical browser packages.

## Remaining release blockers

Manifest-loaded cross-browser E2E, real browser crash injection, Firefox opaque-frame parity, an eight-hour multi-tab soak, manual accessibility, signing, and store review remain `insufficient_evidence` or `not_run`. Therefore `production_ready` remains false.
