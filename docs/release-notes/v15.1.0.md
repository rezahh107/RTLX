# RTLX 15.1.0 — Browser Harmony Hardening

RTLX 15.1 is a reliability-only update over 15.0. It preserves the Persian/English product scope, Profile Schema v3, optional host-permission model, declarative profiles, font policy, and ownership-safe rollback architecture.

## Reliability changes

- bounded online performance aggregates with a maximum 256 raw samples per phase;
- scheduler cleanup across abort, success, idle, MessageChannel, and timeout paths;
- explicit active/passive/hidden/frozen/resumed/destroyed page lifecycle coordination;
- BFCache/prerender guards and bounded resume discovery;
- bounded IntersectionObserver targets and disconnected-node pruning;
- shared delay buckets instead of per-element timers;
- exact generation-scoped owned-mutation suppression;
- deterministic adaptive backpressure and degradation levels 0–4;
- deduplicated/rate-limited diagnostic batching;
- IME-safe input-event coalescing with a bounded caret window;
- transaction-marked, startup-recoverable service-worker storage operations;
- required lifecycle, permission, timer, listener, observer, rollback, and termination stress suites;
- deterministic manifest-loaded browser harness and two-run release packaging evidence.

## Preserved safety

RTLX still never assigns automatic `dir` to `html` or `body`; uses semantic `dir`; keeps Persian RTL/Vazirmatn and English LTR/Inter; does not treat Arabic, Urdu, or Hebrew as Persian; protects code/math/editor/terminal/icon/accessibility zones; never bundles Amazon Ember; and persists no page text, full URL, query, or fragment in diagnostics.

## Verification limits

Development checks, all target builds, direct Chromium runtime smoke, Firefox lint, security scan, production audit, and deterministic packaging pass. Manifest-loaded Chromium is blocked by administrator policy in the execution environment; Edge and Firefox executables are absent. Real BH-013 performance/memory/CLS thresholds, manual accessibility, signing, and store review remain release blockers.
