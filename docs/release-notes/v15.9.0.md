# RTLX 15.9.0 — Effectiveness and Streaming Resilience

## Implemented

- coalesces duplicate and nested streaming mutation roots before processing;
- flushes bounded streaming work at capacity rather than immediately dropping the next root;
- restores queued roots after a failed flush;
- counts one active overflow episode instead of escalating once per rejected root;
- adds condition-based nonterminal degradation recovery after verified runtime quiescence;
- adds degradation transition and dwell-time telemetry;
- adds explicit `ready | partial | blocked` capture readiness;
- adds deterministic build-input and loaded-profile SHA-256 provenance;
- upgrades Runtime Snapshot schema to `1.8.0`;
- adds streaming-resilience, capture-readiness, degradation-recovery, and provenance tests.

## Deliberately unchanged

`profiles/bundled/claude.json` and other site selectors were not rewritten. Aggregate runtime counters and conversational descriptions are insufficient to establish stable production-DOM selectors. A future profile version requires controlled selected-element evidence and positive/negative real-site cases.
