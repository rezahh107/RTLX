# RTLX 15.7.3 — Layout-Safe Direction & Icon Integrity

## Fixed

- Prevented RTL direction from being applied to icon-bearing flex/grid layout containers.
- Separated semantic block, direction target, and typography target.
- Redirected direction to safe text owners or suppressed it when none exists.
- Protected SVG, role-img, PUA, pseudo-element, and icon-font boundaries.
- Allowed safe label typography inside icon-bearing controls without modifying icons.
- Added layout-safety and inherited-direction diagnostics.
- Flushed bounded diagnostics before failure-report export.

## Evidence boundary

The Qwen root cause is supported by a user-provided same-page before/after report. The delivered 15.7.3 build still requires a repeated test on that exact Qwen page before the visual repair can be classified as externally verified.
