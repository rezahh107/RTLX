# RTLX 15.9.7 Release Notes

## Scope

This release hardens cross-browser extension messaging. It does not add user-facing features or alter page-analysis behavior.

## Changes

1. Background responses pass through `enforceResponseMessage()` before `sendResponse`.
2. Canonical failures report `RTLX-MESSAGE-005` rather than transporting invalid values.
3. The shared consumer validator reports safe structural evidence without retaining payload values.
4. Content responses are canonical-checked at both producer and consumer boundaries.
5. Bootstrap reports now use schema `2.0.0`.
6. Chromium exact-artifact evidence verifies context loading after service-worker restart.

## Compatibility basis

Chrome documents extension messages as JSON-serializable. Firefox WebExtensions may preserve values through a different serialization model. RTLX therefore restricts its own protocol to deterministic canonical JSON at both ends.

Official references:

- https://developer.chrome.com/docs/extensions/develop/concepts/messaging
- https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Chrome_incompatibilities
- https://extensionworkshop.com/documentation/develop/getting-started-with-web-ext/

## Evidence status

Deterministic transport tests cover direct objects, JSON round-trips, and structured clones. Real-browser results remain separately classified per target.
