# RTLX 15.5.4 Release Notes

## Summary

RTLX 15.5.4 fixes the activation UX problem confirmed by the user-provided debug report for `chat.deepseek.com`: the page had no persistent host permission, no registered content script, and no reachable content runtime. The popup now exposes an explicit Persian on/off switch for the current page.

## Fixed

- Added a visible per-page activation switch in the popup.
- Turning the switch on asks for optional host permission for the current site from the direct user gesture, saves the enabled site mode, and applies the current tab.
- Turning the switch off disables the current site and sends rollback.
- The current-page debug report action now asks for site permission first when possible, so the next report can include runtime/pageDebug evidence instead of only `permission_missing`.

## Preserved from 15.5.3

- Mixed Persian/English technical content remains typography-eligible.
- The bundled mixed-text/Vazirmatn font remains before broad Windows UI fallbacks.
- The debug report remains text-free and user-initiated.

## Not changed

- No new manifest permission.
- No telemetry or automatic upload.
- No Profile Schema v3 change.
- No full DOM, HTML, page text, form value, cookie, site-storage, console, network, or screenshot capture.
