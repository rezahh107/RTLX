# RTLX 15.9.1 — Runtime Takeover and Evidence Stabilization

## Implemented

- inventories and reconciles pre-existing RTLX-owned DOM state before normal processing;
- claims a document-scoped runtime lease and stops a 15.9.1 runtime that loses ownership;
- adds explicit owner metadata to every new direction mutation;
- preserves ambiguous legacy or host-owned `dir` attributes instead of guessing ownership;
- prunes disconnected text-block, typography-continuation, and protection-reconciliation work;
- waits for bounded runtime stabilization before returning runtime or failure snapshots;
- records startup reconciliation, detached-work pruning, and stabilization evidence;
- upgrades Runtime Snapshot to `1.9.0`;
- adds deterministic unit tests and a controlled Chromium stale-ownership takeover fixture.

## Deliberately unchanged

- language classification and direction thresholds;
- `official:qwen` and `official:claude` selectors;
- streaming queue v2 and degradation policy from 15.9.0;
- permissions and privacy exclusions.

## Verification boundary

Local deterministic checks and the controlled Chromium fixture can verify implementation. Real Qwen/Claude visual effectiveness remains `insufficient_evidence` until the final 15.9.1 browser package is tested on a clean/reloaded page with active streaming, a later `ready` capture, and selected-element evidence for any remaining visible failure.
