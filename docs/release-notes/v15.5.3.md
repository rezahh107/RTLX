# RTLX 15.5.4 Release Notes

## Summary

RTLX 15.5.4 fixes the practical failure mode seen on Persian technical pages where content combines Persian prose with English/CSS terms. It also adds a one-click Persian page debug report so support can diagnose whether the content script ran, whether RTLX owned any candidates, and which font/direction styles were actually computed.

## Fixed

- Mixed Persian/English technical content now enters the typography path instead of being skipped because it is not pure `persian`.
- The bundled mixed-text/Vazirmatn font now has priority over broad Windows UI fallbacks.
- `Segoe UI` and `Tahoma` are no longer part of the Persian local-font alias that could block the bundled font.

## Added

- Persian popup button to download a current-page debug JSON report.
- Text-free runtime `pageDebug` evidence for injected style, bundled font-face presence, owned candidate counts, candidate direction, text alignment, computed font family, and effective settings.

## Not changed

- No new permissions.
- No telemetry or automatic upload.
- No Profile Schema v3 change.
- No full DOM, HTML, page text, form value, cookie, site-storage, console, network, or screenshot capture.
