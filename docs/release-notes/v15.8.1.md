# RTLX 15.8.1 — Resumable Coverage and Verifiable Release

## Fixed

- `RTLX-TEXT-BLOCK-003`: structured responses with more than 512 descendants now continue deterministically until enumeration completes.
- `RTLX-CACHE-001`: typography cache fingerprints now react to protection, ownership, and DOM-context changes.
- `RTLX-RELEASE-001`: browser artifacts and delivery components are hashed only after final creation and are verified against exact paths, sizes, and SHA-256 values.
- `RTLX-EVIDENCE-001`: unexecuted external gates now report `blocked/insufficient_evidence` and exit `2`, never `passed`.
- `RTLX-DEPENDENCY-001`: the vulnerable development-only `undici` transitive version is overridden to `7.28.0`; full and production audits use the high-severity gate.

## Added

- resumable `TextBlockEnumerationCursor`;
- bounded `TypographyProtectionCursor`;
- journal-safe `removeJournaledClass()`;
- Runtime Snapshot `1.7.0` coverage fields;
- deterministic release-integrity generator and verifier;
- boundary, transition, mutation, and external-status regression tests.

## Not certified by this release environment

Exact Edge, Firefox Desktop, Firefox Android, real Qwen 15.8.1 same-page behavior, Claude composer/IME/accessibility, store signing, installed-update rollback, staged rollout, and long-duration soak require external execution and remain `insufficient_evidence`.
