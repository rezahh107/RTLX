# RTLX 15.5.2 — Persian UI, Typography and Auto-Safe RTL Hardening

RTLX 15.5.2 addresses the user-reported issues that Persian output was not reliably right-aligned, Vazirmatn was not visibly applied, and the popup/options UI was too dense and English-heavy.

## Implemented changes

- Default global site mode is now `auto-safe` instead of `ask`.
- High-confidence Persian candidates inherited from an LTR ancestor are now corrected in `auto-safe` mode by setting `dir="rtl"` on the candidate only.
- `html` and `body` remain immutable by automatic processing.
- Typography now prefers local Windows-installed Persian fonts through CSS `local()` before falling back to bundled Vazirmatn.
- The popup was rebuilt as a Persian-first minimal UI with a primary quick action, grouped controls, and advanced tools collapsed by default.
- The options page now includes a Persian quick-start guide, default site-mode control, typography controls, and local-font behavior guidance.
- No `local-fonts` permission was added; the extension does not enumerate or read system font files.
