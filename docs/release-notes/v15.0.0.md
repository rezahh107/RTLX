# RTLX 15.0.0 — Hardening Release

RTLX v15 stabilizes the Persian/English feature set introduced through v14. It does not add another language pack or weaken the frozen SSOT boundaries.

## Added

- Profile Health Engine with healthy/degraded/no-match/excessive/invalid states.
- Streaming Stability Controller with bounded quiet and maximum-wait windows.
- Runtime health, performance, queue, wrapper, root, and journal snapshots.
- Rule Conflict Inspector for accepted and suppressed matches.
- Last Known Good Profile history with canonical SHA-256 entries and safe restore-as-new-version behavior.
- Text-free structural fixture recorder.
- Explicit certification metadata for all twelve bundled profiles.
- Real Chromium content-runtime smoke over the final built bundle.
- Hardening fixtures 34–36.

## Fixed

- Owned typography styles are now appended to `Document.head` instead of directly to `Document`. The previous path raised `HierarchyRequestError` in a real Chromium runtime and prevented typography class and BiDi operations from completing.

## Preserved

- Persian RTL and Vazirmatn.
- English LTR and Inter, with Amazon Ember local-only fallback.
- Smart Picker, per-selector rules, conversation scope, Input Direction Assistant, list repair, context-menu overrides, offline profile library, signed imports, Side Panel/sidebar, diagnostics, journal, and rollback.

## Verification boundary

Unit/property/security checks and a real Chromium content-runtime smoke are automated. Full manifest-loaded cross-browser E2E, live profile certification, manual accessibility, pinned performance/memory, store signing, and store review remain required before production approval.
