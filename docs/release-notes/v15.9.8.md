# RTLX 15.9.8 Release Notes

## Scope

Diagnostic reliability and exact-artifact verification only. No page-analysis or user-feature behavior was intentionally changed.

## Repairs

1. `statusResponseContractViolation` and request-contract failure states keep the popup report action available.
2. A deterministic popup test covers `RTLX-MESSAGE-005`, report enablement, download, schema `2.1.0`, and provenance.
3. Contract issues and bootstrap reports include stable producer, handler, message type, request ID, failure boundary, invalid paths, and invalid value kinds. Healthy response envelopes remain unchanged.
4. Canonical JSON detects cycles and depth greater than 64 and throws deterministic `TypeError` messages using `RTLX-CANONICAL-001` and `RTLX-CANONICAL-002`.
5. Request-side validation rejects non-canonical, oversized, or malformed requests before transport while retaining the original request ID.
6. CI rejects `message_serialization: structured_clone` unless the dedicated structured-clone suite and script are present.
7. Exact-artifact evidence records pass, failure, or insufficient evidence in JSON. Chromium CDP commands and child execution are bounded to prevent indefinite hangs.
8. The content-runtime smoke fixture uses a local HTTP origin rather than `about:blank`, matching the hostname contract used in production pages.

## Deferred

- No lifecycle/restart production-code hardening was added without a qualifying runtime failure trace.
- Firefox Android runtime verification remains unclaimed; its artifact is built and linted only.

## Release state

`NO_GO` for production publication until exact-artifact Chrome and Firefox Desktop evidence is `pass`.
