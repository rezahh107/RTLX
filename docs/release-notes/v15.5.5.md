# RTLX 15.6.0 Release Notes

RTLX 15.6.0 addresses the real large-page symptom reported on Gmail: only a small portion initially changed, more changed after scrolling, and some text never received the intended font.

## Changed

- Visible content remains first priority, but offscreen candidates now drain without user scrolling.
- Discovery continues in deterministic batches beyond the previous first-100-root boundary.
- Profile/semantic content is prioritized over interface buttons and toolbars.
- Persian and mixed Persian/English font styling is applied to protected text-bearing leaves and checked against the computed font.
- Debug reports include queue, discovery, typography, and processing counters.
- Diagnostics are bound to the exact tab/frame/document/runtime before being included in a page report.
- Normal resource pressure cannot permanently pause the runtime at degradation level 4.
- Cooperative yielding uses feature detection and existing fallbacks.

## Unchanged

No new permissions, telemetry, uploads, remote code, page-text capture, full DOM/HTML capture, screenshot capture, or Profile Schema change is included.

## Evidence status

Automated repository verification is recorded in the implementation report. The user’s exact Windows/Gmail result remains `insufficient_evidence` until this build is installed and exercised there.
