# RTLX 15.3.0 — Operational Proof & Update Safety

RTLX 15.3 is a reliability-only update over 15.2. It does not change product scope or Profile Schema v3.

## Added

- persistent update quiescence and update-marker recovery;
- fail-closed content startup during Safe Mode, update pending, or background-context failure;
- persistent Safe Mode after repeated critical failures with verified recovery;
- serialized `storage.sync` writes, conservative rate budgets, canonical read-back, and hash-only conflict evidence;
- bounded runtime-context census through feature-detected `runtime.getContexts()`;
- permission-denial cooldown for policy/user rejection loops;
- dedicated Firefox Android build without desktop-only APIs;
- performance, enterprise-policy, Firefox Android, and release-gate evidence scripts.

## Preserved

Profile Schema v3, Persian/English scope, semantic `dir`, Vazirmatn/Inter policy, Amazon Ember local-only policy, code/math/editor/terminal/icon/accessibility protection, privacy defaults, optional host access, declarative profiles, and journal-safe ownership-checked rollback.

## Executed development evidence

- 69 test files and 176 tests pass.
- Coverage: 85.43% statements, 75.05% branches, 90.20% functions, and 87.03% lines.
- Four browser targets build and validate.
- Direct Chromium runtime smoke, security scan, production audit, and complete dependency audit pass.
- Two release runs produce byte-identical packages.

## Remaining release blockers

Manifest-loaded desktop browsers, Firefox Android device execution, real crash injection, the eight-hour multi-tab soak, the 30-run/browser performance campaign, enterprise-policy execution, manual accessibility, signing, and store review remain `insufficient_evidence` or `not_run`. Therefore `production_ready` remains false.
